# `srxmcp-v0.2.0` — IDP signature-package lifecycle (`manage_idp_security_package`) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the `manage_idp_security_package` MCP tool on `rust-srxmcp`, growing the tool surface from 5 → 6. Ship as `srxmcp-v0.2.0`. AppID equivalent ships separately as `srxmcp-v0.2.1` reusing the shared `signature_package/` submodule landed here.

**Architecture:**
- New shared submodule `rust-srxmcp-core/src/workflows/signature_package/` with four files (`mod.rs`, `preflight.rs`, `poll.rs`, `plan.rs`) carrying the ~60% of logic IDP and AppID share.
- New workflow module `rust-srxmcp-core/src/workflows/idp_package.rs` with verb dispatch (`check_server`, `download_and_install`, `rollback`) and an IDP-specific RPC dispatch table.
- 9 new `SrxError` variants in `rust-srxmcp-core/src/error.rs`, each carrying `[code=<snake>]` in Display (matching `JmcpError`'s convention). (A5: `commit_confirmed_active` dropped — handled via non-blocking audit warn instead.)
- New `#[tool]` method `manage_idp_security_package` added inline to `JmcpSrxHandler` in `rust-srxmcp/src/server.rs`, matching the Phase 1B inline pattern.
- Per-router serialization via `TransferLocks` (from `rust-junosmcp-core/src/tools/transfer_file.rs`), acquired BEFORE call-2 pre-flight re-runs.
- Audit lines emitted via `tracing::info!(target = "audit", ...)`, matching `upgrade_junos`. No new on-disk JSONL file.

**Tech Stack:** Rust 2021, tokio, rmcp 0.8, rustez 0.12 / rustnetconf 0.12, quick-xml, schemars, serde, thiserror, tracing.

**Reference spec:** `docs/superpowers/specs/2026-05-22-srxmcp-phase-2-signature-packages-design.md`

**Reference modules to imitate:**
- Two-call confirm + per-router lock + audit guard pattern: `rust-junosmcp-core/src/tools/upgrade_junos.rs`
- Phase 1B inline `#[tool]` registration: `rust-srxmcp/src/server.rs`
- `SrxError`-style workflow result handling: `rust-srxmcp-core/src/workflows/cluster_status.rs`

## Pre-implementation decision to lock down

**A5 (from eng review): resolved 2026-05-25 — DROP the `commit_confirmed_active` blocker.**

Sig-package install is op-mode (`request security idp security-package install`).
Binaries land in `/var/db/idpd/` outside the candidate-config / commit pipeline,
so there is no `mgd` codepath where a commit-confirmed window interacts with the
install. `upgrade_junos` keeps the blocker because OS upgrades touch loader-env
and boot-config; sig-packages don't. Phase 1B tools (services_status, license,
vpn_lifecycle, cluster_status) don't check commit-confirmed either — keeping it
for IDP would be the outlier.

Non-blocking alternative implemented in Task 3: pre-flight detects an active
commit-confirmed window and emits `tracing::warn!(target = "audit", event = "sigpkg_commit_confirmed_window_active", ...)`,
then returns Ok. Audit trail captures the unusual condition without refusing
the call.

**Net effect on this plan:** Task 2 lands **9** new variants, not 10. The
`SignaturePackageCommitConfirmedActive` variant is dropped. Task 1 still
captures the `commit_confirmed_active.xml` fixture because the parser is
reused (just for the warn classifier rather than a blocker).

  ```rust
  #[tool(
      name = "manage_idp_security_package",
      description = "DESTRUCTIVE on the `download_and_install` and `rollback` actions: ..."
  )]
  async fn manage_idp_security_package(
      &self,
      Parameters(args): Parameters<rust_srxmcp_core::IdpPackageArgs>,
      extensions: Extensions,
  ) -> Result<CallToolResult, rmcp::ErrorData> {
      let mut device = self.device_manager.open(&args.router).await
          .map_err(|e| rmcp::ErrorData::internal_error(format!("opening device: {e}"), None))?;
      let resp = rust_srxmcp_core::workflows::idp_package::run(&mut device, args).await
          .map_err(|e| match e {
              SrxError::InvalidInput(_) => rmcp::ErrorData::invalid_params(e.to_string(), None),
              SrxError::SignaturePackageConfirmationRequired { .. } => {
                  rmcp::ErrorData::invalid_request(e.to_string(), None)
              }
              _ => rmcp::ErrorData::internal_error(e.to_string(), None),
          })?;
      let body = serde_json::to_string_pretty(&resp).map_err(|e| {
          rmcp::ErrorData::internal_error(format!("serializing IdpPackageResponse: {e}"), None)
      })?;
      Ok(CallToolResult::success(vec![Content::text(body)]))
  }
  ```
